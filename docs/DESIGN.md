# VoxBrush 声笔 · 设计文档

> **七牛云校招挑战 · 题目二：AI 语音绘图工具**
> 线上地址：<https://qiniu.versecraft.cn> ｜ GitHub: `bei666qi-pan/voxbrush` ｜ Gitee 镜像同步
> 业内调研：见 [docs/INDUSTRY-RESEARCH.md](./INDUSTRY-RESEARCH.md)

**核心约束：用户不能使用鼠标或键盘，仅通过语音完成绘图创作。**
评价维度：指令理解准确性与容错性 / 语音→绘图响应延迟 / 复杂指令拆解与执行能力。

## 0. 业界方案对比（为什么选择混合架构）

在启动本项目前，我们深度调研了业界四种主流技术路线（详见 `docs/INDUSTRY-RESEARCH.md`）：

| 路线 | 代表项目 | 核心机制 | 延迟 | 可编辑 | 复杂指令 |
|------|---------|---------|:---:|:---:|:---:|
| A. 纯 AI 文生图 | studiBudi Canvas (Gemini/Stability) | ASR→LLM→图像生成模型→位图 | 5-12s | ⛔ | ✅ |
| B. Agent + 组件绘制 | Agentic Voice Illusion (CrewAI+Excalidraw MCP), **VoxBrush** | ASR→LLM Agent→绘图 DSL→场景图 | 0.5-3s | ✅ | ✅ |
| C. 规则匹配 | Voice-Activated Drawing Canvas, VoxCanvas | ASR→正则匹配→直接绘图 | <5ms | ✅ | ⛔ |
| D. 多模态空间 | SpatialPrompt (Apple Vision Pro) | 语音+3D笔+AI 3D生成 | 实验性 | ⚠️ | ✅ |

**结论**：路线 A 画质好但不可迭代编辑；路线 C 快但只能处理固定句式；路线 D 硬件门槛高。
**VoxBrush 采用 B+C 混合架构**：本地解析器覆盖 85% 高频指令（<5ms），LLM Agent 处理复杂场景拆解，
统一绘图 DSL 保障可编辑性与确定性——是唯一同时满足「低延迟 + 可编辑 + 复杂指令 + 国内可用」四重约束的方案。

## 2. 关键技术决策与权衡

### 2.1 ASR：自托管流式模型，而非云 ASR API
- **决策**：服务端内嵌 [sherpa-onnx](https://github.com/k2-fsa/sherpa-onnx) 流式 Zipformer（中英双语，int8 量化），WebSocket 推流，边说边出字，自带端点检测（VAD endpointing）。
- **理由**：
  1. *延迟*：模型与业务同机，无公网 RTT，部分结果 <150ms 返回；
  2. *国内网络*：零外部依赖，不受跨境网络影响（浏览器 Web Speech API 在国内 Chrome 不可用）；
  3. *成本/凭证*：火山语音技术产品线需单独开通 AppID，当前凭证仅有 AK/SK；
  4. *权衡*：自托管模型准确率略低于云端大模型 ASR —— 用「绘图领域热词后处理 + 容错归一化（同音纠错）」弥补。
- **兜底**：浏览器 Web Speech API 作为可选引擎（设置中切换），互为降级。

### 2.2 指令理解：三级流水线（速度优先，能力兜底）
| 层 | 引擎 | 延迟 | 覆盖 |
|----|------|------|------|
| L0 | 本地语法解析器（领域 CFG + 中文数词/颜色/方位词法）| <5ms | 单步绘制、修改、选择、画布控制（~85% 高频指令）|
| L1 | 豆包 Seed-1.6-flash · Function Calling | ~0.6–1.5s | 复合指令拆解（"画一座房子，左边一棵树，天上一个太阳"→ N 个原语 Op）|
| L2 | 豆包多模态（画布快照 + 场景 JSON）| ~1–2.5s | 视觉指代消解、构图建议、"评价一下我的画" |

- L0 未命中才升级 L1/L2，**绝不让简单指令付出 LLM 延迟**；
- L1/L2 输出统一收敛为**绘图 DSL（§3）**，经 JSON Schema 校验后执行，杜绝模型幻觉破坏画布；
- LLM 不可用时优雅降级：L0 仍完整可用，语音播报提示。

### 2.3 Ark 鉴权：AK/SK 动态铸钥
仅持有 VOLC_AK/SK。服务端实现火山 SigV4 签名，调用 `open.volcengineapi.com` `Action=GetApiKey` 铸造临时 Ark API Key（带 TTL，过期前自动续期），失败时回退多种 ResourceType 组合并自动探测可用模型 ID（`/api/health` 可见）。`ARK_API_KEY` 环境变量可直接覆盖。

### 2.4 绘图引擎：保留模式场景图 + 命令栈
- 场景图（对象有 id/名字/属性）而非位图：支持"把**那个红色的圆**改成蓝色"这类对象级语音编辑；
- 全部变更走 Command 栈：天然支持 撤销/重做/宏回放；
- 渲染：Canvas2D，requestAnimationFrame 批量重绘，新增对象带 200ms 缩放入场动画（语音操作的视觉确认）。

### 2.5 国内部署工程
- Docker 多阶段构建：npm 走 npmmirror；ASR 模型从 hf-mirror.com 下载（GitHub Releases 兜底）；
- Coolify 拉取 **Gitee** 仓库（火山 ECS 拉 GitHub 不稳定），GitHub 为主仓库、Gitee 自动同步。

## 3. 绘图 DSL（语音 → 操作 的统一契约）

```jsonc
{ "op": "add", "shape": "circle|ellipse|rect|...|path|sprite|image",
  "props": { "x":400,"y":300, "asset":"house", "palette":["#f5deb3","#c0392b"],
             "gradient":{"type":"linear","stops":[[0,"#87CEEB"],[1,"#E0F6FF"]],"angle":180},
             "shadow":{"blur":8,"color":"#00000033","dx":2,"dy":4}, "z":10, "d":"M...", ... } }
{ "op": "generate_image", "prompt":"...", "as":"background|object", "w":960, "h":600 }
{ "op": "modify", "target": {...}, "set": {"fill":"#3498db"}, "delta": {"dx":50,"scale":1.25,"rotate":30} }
{ "op": "select" | "delete", "target": { "kind":"last|selected|name|query", "shape":"sprite", "color":"red", "position":"left" } }
{ "op": "undo" } { "op": "redo" } { "op": "clear" } { "op": "background", "color":"#0b1021" }
{ "op": "save" } { "op": "say", "text": "好的" } { "op": "help" }
```

L0/L1/L2 三层输出同一 DSL；前端执行器是唯一事实来源。

> 以上未完成项均不影响核心创作闭环。现已实现能力覆盖从基础形状到 AI 辅助创作（评画/美化/文生图/风格渲染）的全链路。

**生图链路打通记录（PR #4，已解决）：** 早期「生图鸡肋」其实是**两个被误判为鉴权的真问题**叠加，逐一定位修复：
1. **模型 ID 错误**：代码硬编码 `doubao-seedream-4-0-250415`，但本账号实际可用的是 `doubao-seedream-4-0-250828`，旧 ID 返回 `InvalidEndpointOrModel.NotFound`；
2. **尺寸约束**：Seedream 4.0 要求图像单边 ≥768，而代码用了 `512x512`/`1024x576`，返回 `InvalidParameter`；
3. **误判放大**：`isAuthErr` 的通用 `invalid` 把上面两类参数错误误判为鉴权失败，触发无谓的 SigV4 直签兜底，最终把错误掩盖成「API key invalid」。

修复后（model→`250828`、size→`1280x800`(8:5 贴合画布)/`1024x1024`、`isAuthErr` 去掉通用 `invalid`、持账号级 `ARK_API_KEY` 时直用模型名）：**文生图与 img2img 风格渲染均线上实测可用**（`/api/health`→`ark.imageAuthOk:true`，`/api/render`→`mode:img2img`）。

工程化的诚实兜底仍保留：启动真实探测一次生成写入 `imageAuthOk`；`imageModelAvailable()` 据此返回，**鉴权/模型不可用时 L1 自动移除 generate_image/render_style**（红线 #5），render_style 的 L0 指令仍触发但优雅降级，矢量链路永不受影响。凭证只存于运行时环境变量（`ARK_API_KEY`），不入库。

## 5. 延迟预算（设计目标 / 实测值随版本更新）

| 阶段 | 预算 | 说明 |
|------|------|------|
| 音频采集→服务端 | <60ms | AudioWorklet 20ms 帧 + WS 二进制 |
| 流式 ASR 部分结果 | <150ms | int8 Zipformer，同机推理 |
| 端点检测(说完→final) | 300–500ms | trailing silence 可调 |
| L0 解析+渲染 | <16ms | 一帧内完成 |
| **简单指令端到端** | **<0.7s** | 说完即画 |
| L1 复杂拆解端到端 | 1.5–3s | flash 模型 + 受限 JSON 输出 |

**线上实测（v1.3 · PR #3）：**
- L0 单素材指令（「画一棵树」）解析+渲染 **<5ms**（HUD 可见）；
- L0 克隆指令（「再来一棵」）解析+渲染 **<3ms**（纯本地数据操作，比 L1 快 1000x）；
- L0 宏回放（「画一个小屋」）解析+渲染 **<5ms**（含 3~8 个对象批量 add）；
- L1 复合场景（「画一座小村庄：两座房子一棵树，天上有彩虹」）SSE 首笔 **~1.2–1.8s**；
- L2 评画（「评价一下我的画」）端到端 **~2–4s**（含画布快照编码+多模态推理）；
- L2 美化（「帮我美化构图」）SSE 首笔修改 **~2–4s**，逐笔可见；
- 17 种 sprite 本地矢量绘制，零网络延迟；
- Seedream 文生图：`/api/image` 可选增强，端点不可用时不阻塞核心链路（`/api/health` → `ark.imageAvailable`）。

**线上实测（v1.4 · PR #4 风格渲染 + 语音纠错）：**
- 领域同音纠错（「申屠成水彩风」→「生图成水彩风」→命中 render_style）解析 **<1ms**，纯本地正则；
- ASR 热词层默认启用 modified_beam_search（`/api/health`→`asr.decodingMethod`），构造失败自动回退 greedy；
- 风格化渲染 `/api/render` img2img 端到端 **~8–15s**（Seedream 推理为主），期间字幕「正在渲染…大约十秒」+ thinking HUD 防冷场；img2img 不可用时降级文生图，矢量层始终不阻塞；
- 渲染后矢量对象仍可「把房子改成蓝色」「撤销」（渲染层 z=-50，矢量 z≥0）。

**v1.2 基线（PR #2）：**
- L0 单素材指令（「画一棵树」）解析+渲染 **<5ms**（HUD 可见）；
- L1 复合场景（「画一座小村庄：两座房子一棵树，天上有彩虹」）SSE 首笔 **~1.2–1.8s**；
- 17 种 sprite 本地矢量绘制，零网络延迟；
- Seedream 文生图：`/api/image` 可选增强，端点不可用时不阻塞核心链路（`/api/health` → `ark.imageAvailable`）。

**v1.1 基线：**
10 元素山水画整句拆解 ~6.6s（非流式全量）；v1.1 引入 SSE 流式逐笔绘制后，
首笔在模型生成出第一个完整 op 时即落画布（实测 ~1–2s），用户感知延迟大幅下降。
教训记录：seed-1.6 系列默认开启深度思考，交互式场景必须显式 `thinking: disabled`，
否则单次拆解 >30s 直接超时 —— 这是"延迟与效果权衡"中影响最大的一个开关。

HUD 实时显示各阶段耗时（ASR / 首笔 / 全量），做到延迟可观测。

## 6. 容错设计
1. **词法容错（双层 ASR 纠错）**：
   - *服务端 ASR 热词*（`server/src/asr.js`）：绘图领域指令/素材/风格词做 sherpa-onnx 热词表（modified_beam_search，hotwordsScore 2.0），在解码阶段提高生僻词召回；构造失败自动回退 greedy_search，绝不拖垮 ASR。
   - *前端领域同音纠错*（`parser.ts` `correctDomain`）：把「申屠/生土/声图→生图」「水菜→水彩」「渲图→渲染」「记不力/吉布力→吉卜力」等高频误识别纯本地正则归一，<1ms，独立于热词生效，是最稳的兜底；HUD 展示「原文→纠错后」。
   - 另含同音字归一（园→圆）、口语量词剥离。
   - *权衡记录*：负向后瞻 `(?<!渲)染成` 保护「渲染成X」不被既有「染成→改成」规则误伤。
2. **语义容错**：缺省参数补全（未说位置→画布空白区自动布局；未说颜色→主题色轮换）；
3. **执行容错**：DSL Schema 校验 + 越界钳制；目标选择无歧义匹配失败时语音追问；
4. **链路容错**：LLM 超时/不可用→L0 降级+播报；WS 断线自动重连；ASR 引擎可切换；
5. **误触发抑制**：能量 VAD + 最短语句长度过滤背景噪声。

## 7.5 工程实战记录（国内网络）
部署过程中实际踩到并解决的问题，均已固化进仓库：
docker.io 拉取停滞 → 基础镜像切 `docker.m.daocloud.io`；deb.debian.org apt 缓慢 →
模型下载改为 Node fetch 零 apt 依赖；ASR 模型走 hf-mirror；npm 走 npmmirror；
Gitee API 建仓默认私有 → PATCH 转公开；`GetApiKey` 需绑定真实推理接入点 →
服务启动时 `ListEndpoints`/`CreateEndpoint` 自举；域名解析缺失 →
启动时经火山云解析 OpenAPI 自动注册 A 记录（DNS 自举）。
